Laptop 安装 Kali Linux

Posted on 2017-02-13 | In Diary

前言

周一至周五上班，双休日在家。Python 已经好几天没学了，《Byte Of Python》 看到 模块 部分，
打算这个月结束前把 《Byte Of Python》 看完，3月份就可以学习写个爬虫了。

前几个礼拜把Kali Linux安装到一个 8G 的 Sandisk U 盘，upgrade发现空间不太够。就把弟弟遗落在送我的背包里的PNY 32G 拿出来格式化分区安装。

Kali linux

今天收到前几天在淘宝买的 Sandisk CZ80 32GB ，就按照网上教程安了。

  • 先用DiskGenis分区，因为将 ISO 刷到 U 盘只需要4.2GB的空间，所以先分了一个6GB大小的主分区。
  • 用 Universal-USB-Installer 配置成 Kali Linux 并找到 ISO 镜像目录，Format Fat 32 刷入。
  • Tips : 余下的空间不格式化。
  • reboot 电脑，Bios 用 Legecy 引导 U 盘启动。
  • 进入安装画面，我尝试过 Kali perisistence 做一个 Kali Live 然后mount /dev/sdc2 分区来储存 conf 文件及数据来使用，效果一般。

先用Gparted分一个Label叫persistence的Ext4分区

安装到U盘

  • 选择 Graphical Install ，其他不说了重要的是手动分一个 4GB大小的（ 大小随意 ） Swap 交换空间，余下的全部作为 EX4格式 Root 目录使用。
  • 安装完毕后进入。

我的 Laptop 比较特殊，有一块 Intel 集成显卡和一块 GTX 970M 3G 版独显。
所以在 Kali 更新完后登陆 Root 用户会卡死在登陆界面。

安装完后，第一时间将 Firefox 的 baidu search add-on install 后更换 Google 默认搜索引擎。

解决方法

翻了很多帖子，确定了是双显卡问题，然后在百度帖吧看到我的 MSI Laptop 电源键上的指示灯黄色时候是独显在工作，蓝色为集显在工作。我使用 Kali 的时候集显跟核显居然一起开启。
而更新了 APT 后，Kali 会加载 nouveau 这个第三方开发的 Nvidia 驱动（罪魁祸首）。

卡死后强制关机重启，先到Grub界面，网上有在 quiet 替换成 nomodeset quiet plash 方式进安全模式的 Kali.
